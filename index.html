This Proof-of-Concept demonstrates how your browser can process traffic light information. The intersection at the Tropic Institute of Antwerp is upgraded to a smart traffic light for the City of Things project. This data has been published as Linked Open Data enabling user agents to process this data more intelligent.

How does it work?

Traffic light updates can change very frequently (every few seconds) and contains little information per update. As HTTP/1.1 requires the client to re-establish a TCP connection for every GET resource, new trade-offs need to be found. One way of solving this is by using publish-subscribe technologies. Another way is leveraging upcoming HTTP/2 technology where the same TCP connection can be used for retrieving multiple resources. This project proposes 3 types of interfaces:

  • HTTP/2 documents where each document represents a resource and can be cached for multiple seconds enabling low-cost data publishing. Try it out.
  • Server-Sent Events interface creating a mechanism to subscribe for updates and pushing emergencies to the consumers
  • Historic data contains updates fragmented together in HTTP documents for every resource (on traffic light or intersection level)
